## 1. 시스템 개요

### 1.1 아키텍처

```text
[Browser]
  FileUpload → Papa Parse → useEDA (stats + charts) → Zustand store
                                                      ↓
  DataSummary / ChartViewer / InsightPanel ←─────────┘
                                                      ↓
  InsightPanel → POST /api/insights (axios) ──→ [Vercel Edge] api/insights.ts
                                                      ↓
                                            OpenAI Chat Completions (gpt-4o-mini)
```

- **클라이언트**: CSV 파싱, 통계 요약, 차트 스펙 생성 전부 브라우저에서 수행.
- **서버**: `/api/insights`만 Edge 함수로 존재하며, OpenAI 호출에 사용. 키가 없으면 JSON으로 안내 메시지를 반환.

### 1.2 배포

- `vercel.json`: `api/*.ts`에 **edge** 런타임 지정.

## 4. 데이터 모델

### 4.1 `DataRow`

```ts
Record<string, string | number | null>
```

### 4.2 `ColumnType`

`numeric` | `categorical` | `datetime` | `unknown`

- **numeric**: 정리된 값이 모두 유한 실수로 해석될 때.
- **datetime**: 수치가 아니고, 문자열의 약 **80%** 이상이 `Date.parse` 성공 시.
- **categorical**: 그 외 문자열 기반.
- **unknown**: 코드상 기본 분기는 categorical/datetime 쪽으로 수렴; 타입은 확장 여지용.

### 4.3 `ColumnSummary`

| 필드 | 설명 |
|------|------|
| `column` | 컬럼명 |
| `type` | 위 타입 |
| `count` | 행 길이(원본 셀 수) |
| `missing` | null/undefined/빈 문자열 개수 |
| `unique` | 고유값 수 |
| `numericStats?` | min, max, mean, median, stdDev |
| `categoryStats?` | `topValues` 최대 5개 (value, count) |

### 4.4 `ChartSpec`

| 필드 | 설명 |
|------|------|
| `id` | 고유 문자열 |
| `title` | 차트 제목 |
| `type` | `histogram` \| `bar` \| `scatter` \| `line` |
| `data` | Recharts에 넘길 행 배열 |
| `xKey`, `yKey` | 데이터 키 |

## 5. CSV 파싱 (`useCSVParser`)

Papa Parse 옵션:

| 옵션 | 값 |
|------|-----|
| `header` | `true` |
| `skipEmptyLines` | `true` |
| `dynamicTyping` | `true` |

결과는 `Promise<DataRow[]>`로 resolve; 에러 시 reject → FileUpload에서 일반 오류 메시지.

**업로드 제한 (UI)**

- 확장자: `.csv`
- 크기: **≤ 10MB**

## 6. 통계·타입 추론 (`stats.ts`)

- 결측: `null`, `undefined`, `''` 제외 후 `missing` 계산.
- 수치: `toNumber`로 전부 유한 숫자면 `numeric`.
- 날짜: 비수치이면서 문자열의 80% 이상이 `Date.parse` 유효면 `datetime`.
- 범주: 빈도 맵 후 상위 5개를 `categoryStats.topValues`에 저장.

---

## 7. 차트 생성 (`charts.ts`)

### 7.1 공통

- 수치 히스토그램: bin 수 `min(12, max(6, floor(sqrt(n)))))`.
- 컬럼명이 `y` 또는 대소문자 무관 `target`인 수치 컬럼 히스토그램: **99분위 클리핑** 후 구간화(극단 꼬리 완화).

### 7.2 생성 규칙 요약

| 조건 | 생성 |
|------|------|
| 수치 컬럼 상위 2개 | 각각 히스토그램 |
| 범주 컬럼 상위 2개 | 막대(상위 10 레이블) |
| 수치 ≥ 2개 | 첫 두 수치 컬럼 산점도(최대 300점) |
| datetime 1개 + 수치 1개 이상 | 시계열 라인(날짜×첫 수치, 최대 240점) |
| 선호 타깃(`y` 또는 `target` 수치) 존재 | 정렬 타깃 프로필 라인(선두 삽입), 첫 범주×타깃 평균 막대 |

최종 배열은 **`slice(0, 6)`** 으로 최대 6개.

### 7.3 `ChartViewer` 렌더링

- `histogram`·`bar`: `BarChart` + `Bar`
- `line`: `LineChart` + `Line`
- `scatter`: `ScatterChart` + `Scatter`
- 높이 240px, `ResponsiveContainer` width 100%.

## 8. 인사이트

### 8.1 API (`api/insights.ts`)

- **런타임**: Edge (`export const config = { runtime: 'edge' }`).
- **요청 본문**: `{ summary: ColumnSummary[], charts: ChartSpec[] }`.
- **환경 변수**: `API_SECRET_KEY` — OpenAI Bearer 토큰. 없으면 HTTP 200으로  
  `insights: ['API key is missing...']` 반환.
- **OpenAI**: `https://api.openai.com/v1/chat/completions`
  - 모델: `gpt-4o-mini`
  - `temperature`: 0.2
  - `response_format`: `{ type: 'json_object' }`
  - 시스템: 데이터 애널리스트, **JSON 배열 형태의 문자열 인사이트** 요청.
  - 사용자: `task`, `summary`, `charts` JSON 문자열.
- **응답**: `{ insights: string[] }`. 본문 파싱 실패 시 단일 문자열로 감쌈.

### 8.2 클라이언트 (`useInsights` + `InsightPanel`)

- `axios.post<InsightResponse>('/api/insights', payload)`.
- 실패 시 `generateHeuristicInsights(summary)` 반환.
- 성공해도 `insights`가 비면 휴리스틱으로 채움.

### 8.3 휴리스틱 (`insights.ts`)

- 결측 비율 > 20% 컬럼 강조, 없으면 “결측 없음”류 메시지.
- 수치 중 표준편차 최대 컬럼 → 분산·이상치 점검 권장.
- 범주 최빈값 편향 → 클래스 불균형 가능 언급.
- 수치 컬럼 2개 이상 → 산점·상관 분석 가능.
- 고유값 ≤ 2인 컬럼 개수 → 이진/준상수 피처 언급.
- 최대 **4개** 문자열로 잘라 반환.

## 12. 보안·운영 참고

- CSV는 기본적으로 **클라이언트 메모리**에만 상주; 인사이트 요청 시 **요약·차트 메타**만 API로 전송.
- Edge 함수는 **요청당** OpenAI 비용·레이트 리밋 발생 가능 → UI에서 수동 버튼으로 호출 제한.
- 프로덕션에서는 CORS·인증·요청 크기 제한 등을 정책에 맞게 추가 검토.
