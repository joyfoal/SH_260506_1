## 1. 배경 및 목적

### 1.1 배경

- 탐색적 데이터 분석(EDA)은 모델링·리포팅 전에 반복적으로 수행되나, 매번 노트북이나 BI 도구를 열어 동일한 패턴의 요약·차트를 만드는 데 시간이 든다.
- 소규모 CSV를 빠르게 올리고 **요약 통계·기본 차트·텍스트 인사이트**까지 한 화면에서 확인할 수 있으면 의사결정 속도가 빨라진다.

### 1.2 목적

1. **단일 CSV 업로드만으로** 컬럼 타입 추론, 결측·고유값, 수치 기초통계를 자동 산출한다.
2. 데이터 성격에 맞는 **차트를 자동 생성**해 시각적으로 패턴을 파악한다.
3. 선택적으로 **AI 기반 인사이트**(또는 API 실패 시 규칙 기반 대체)를 제공한다.

### 1.3 비목표 (이번 범위에서 하지 않음)

- 대용량 데이터 스트리밍·서버 사이드 분산 처리
- 사용자 계정·협업·버전 관리
- SQL/DB 직접 연결, 스케줄 리포트 발송

---

## 2. 타깃 사용자

| 구분 | 니즈 |
|------|------|
| 데이터 분석가·ML 엔지니어 | 샘플 CSV 빠른 스캔, 컬럼 품질·분포 감 잡기 |
| 기획·운영 | 수치·카테고리 비중을 코드 없이 확인 |
| 학습·데모 | EDA 파이프라인(파싱 → 요약 → 시각화 → 인사이트) 이해 |

---

## 3. 핵심 가치 제안

- **설치 없이 브라우저**에서 CSV만 선택하면 분석이 시작된다.
- **클라이언트 EDA**로 민감 데이터를 서버에 올리지 않고도 요약·차트까지 처리 가능(인사이트 API는 선택 사항).
- **초기화**로 세션을 깨끗이 비우고 다른 파일로 전환하기 쉽다.

---

## 4. 기능 요구사항

### 4.1 필수

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | CSV 업로드 | `.csv`만 허용, 최대 **10MB** |
| F-02 | 자동 파싱 | 헤더 행, 빈 줄 스킵, 동적 타입 변환 |
| F-03 | 컬럼 요약 | 타입(numeric / categorical / datetime / unknown), 결측, 고유값, 수치 시 min·max·mean·median·std, 범주 시 상위 빈도 |
| F-04 | 자동 차트 | 히스토그램·막대·산점·라인 등 데이터에 맞게 최대 6개까지 생성 |
| F-05 | 화면 구성 | 업로드 → 개요 테이블 → 차트 그리드 → AI 인사이트 순 |

### 4.2 선택

| ID | 기능 | 설명 |
|----|------|------|
| O-01 | AI 인사이트 | 버튼 클릭 시 서버 API 호출, **3~5개** 수준의 짧은 인사이트 |
| O-02 | 폴백 | API 오류·미설정 시 **규칙 기반** 인사이트로 대체 |

### 4.3 제약·검증

- 비 CSV 확장자·용량 초과 시 사용자에게 메시지 표시
- 파싱 실패 시 오류 메시지 표시

---

## 5. 사용자 시나리오

1. 사용자가 메인 화면에서 CSV를 선택한다.
2. 시스템이 파싱 후 EDA 요약과 차트를 즉시 갱신한다.
3. 사용자가 **인사이트 생성**을 눌러 AI(또는 폴백) 문구를 확인한다.
4. 다른 파일로 바꾸려면 **초기화** 후 다시 업로드한다.
